전통적으로 검증된 APM으로 만든 웹페이지

APM은 Apache + PHP + MySQL 의 약자입니다. 많은 사이트에서 APM을 통해서 웹사이트를 개발하고 유지보수하며 검증 된 Web development Pool을 가지고 있습니다.

  • REGISTER

기술 스택 소개

Apache

PHP 소스파일을 구동시키기 위한 웹서버 입니다.

MySQL

데이터를 저장하기 위한 관계형 데이터베이스 입니다. 저희 수업의 주 내용 이기도 합니다.

PHP

웹 브라우저와 데이터베이스를 연결하고 컨트롤하기 위한 백엔드 언어 입니다. 중소규모의 웹사이트에서 주로 사용합니다.

Git / Github

여러 사람과 함께 업무를 협업하고 Source를 일괄적으로 관리하기 위한 툴입니다.

Visual Studio Code

MicroSoft에서 만든 오픈소스형 소스 에디터 툴입니다. 매우 가볍고 범용적으로 사용가능합니다.

Google Clude

Google 에서 지원하는 클라우드 플랫폼 입니다. 데이터 베이스와 웹서버를 구동 시킬수 있습니다.

과제 내용

다음 조건을 만족하는 데이터베이스와 웹페이지를 구현하라
▪ 참고 사이트: http://dfchallenge.org/
▪ 팀 단위 회원가입
• 최대 6인 회원가입 가능
• 각 팀별로 회원가입 시 필요한 항목은 http://dfchallenge.org/register/ 에서 확인 가능
▪ 랭킹 공개
• DB에 각 팀의 문제 별 점수가 기록
° 문제 종류: 카테고리별 100, 200, 300, 400, 500점 문제
° ex) AF100, AF200, AF300, AF400, AF500
• 카테고리별 점수를 공개하고 총점을 기준으로 순위 매김
http://dfchallenge.org/ranking-2019/
• 각 팀 별로 획득한 점수 공개
ex) http://dfchallenge.org/df-team2019/?Team=noname

과제 분석

1) 회원가입 화면
요구사항 1) 팀 단위 회원가입
요구사항 2) 최대 6인 회원가입
-> 유동적으로 6인 회원가입이나 늘어나거나 줄어드는 경우를 고려할것

2) 랭킹 공개 화면
요구사항 1) 각 팀의 문제별 점수 기록
요구사항 2) 카테고리별로 100점, 200점, 300점, 400점, 500점 짜리 문제로 구성되어있음
-> ex) AF100, AF200, AF300, AF400, AF500
요구사항 3) 카테고리별 점수를 공개하고 총점을 기준으로 순위 매김
오구사항 4) 각 팀 별로 획득한 점수 공개

3) 랭킹 입력 화면 (추가 개발)(관리자기능)

4) 로그인(추가 개발)
